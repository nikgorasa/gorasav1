## Recommended Modules (Priority Order)

### Module 1: Search Filters (P1)

**Why:** When the AI Planner recommends hotels/flights, users need to filter results by price, stars, amenities.

**Files to Create:**
```
components/HotelFilterPanel.tsx    ← Price range, star rating, amenities
components/FlightFilterPanel.tsx   ← Stops, airlines, departure time
hooks/useFilters.ts               ← Filter state management
```

**Integration Point:**
- Planner's `ItineraryPreview` shows "View Hotels" / "View Flights" buttons
- Click opens filter panel → results displayed

---

### Module 2: Multi-Language Support (P1)

**Why:** Indian market needs Hindi support. LLM handles translation natively.

**Files to Create:**
```
lib/ai/i18n/translations.ts       ← Hindi translations for UI strings
lib/ai/i18n/languageDetector.ts   ← Detect user language from input
components/LanguageSwitcher.tsx    ← Language toggle in navbar
```

**Integration Point:**
- HolidayPlanner detects language → responds in same language
- Quick replies translated
- Itinerary titles in user's language

---

### Module 3: Chat Session Persistence (P1)

**Why:** Users abandon planning mid-way. Need to save and resume.

**Files to Create:**
```
lib/ai/session/sessionManager.ts  ← Save/load sessions to localStorage + Supabase
components/ResumeSessionBanner.tsx ← "Continue planning?" banner
app/api/ai/sessions/route.ts      ← GET/POST sessions
```

**Database:**
```sql
CREATE TABLE "PlanningSession" (
  id TEXT PRIMARY KEY,
  userId TEXT,
  messages JSONB,
  itinerary JSONB,
  status TEXT DEFAULT 'ACTIVE',
  createdAt TIMESTAMP DEFAULT NOW()
);
```

---

### Module 4: Agent Handoff Dashboard (P1)

**Why:** When planner hands off to human, admin needs to see full context.

**Files to Create:**
```
app/admin/ai-leads/page.tsx       ← Dashboard for AI-initiated leads
components/AiLeadCard.tsx         ← Lead card with conversation history
components/AiLeadDetail.tsx       ← Full conversation + itinerary view
```

**Integration Point:**
- `HandoffModal` creates lead with `source: "AI_PLANNER"`
- Admin dashboard filters by source
- Agent sees full conversation + itinerary

---

### Module 5: Proactive Assistance (P2)

**Why:** Re-engage users who abandoned planning. Trip reminders before travel.

**Files to Create:**
```
lib/ai/proactive/tripReminder.ts  ← 3 days before departure
lib/ai/proactive/priceAlert.ts    ← Price drop notifications
lib/ai/proactive/reviewRequest.ts ← Post-trip review
components/ProactiveBanner.tsx    ← Notification banner
app/api/ai/notifications/route.ts ← CRUD for notifications
```

**Database:**
```sql
CREATE TABLE "AINotification" (
  id TEXT PRIMARY KEY,
  userId TEXT,
  type TEXT,
  title TEXT,
  message TEXT,
  actionUrl TEXT,
  read BOOLEAN DEFAULT false,
  createdAt TIMESTAMP DEFAULT NOW()
);
```

---

### Module 6: Analytics Dashboard (P2)

**Why:** Measure if planner is converting. Optimize prompts based on data.

**Files to Create:**
```
lib/ai/analytics/tracker.ts       ← Event tracking
app/admin/ai-analytics/page.tsx   ← Dashboard with charts
components/AiMetricsCard.tsx      ← KPI cards
components/AiConversionChart.tsx  ← Conversion funnel
```

**Events to Track:**
- `planner_started` — User opened planner
- `planner_message_sent` — User sent message
- `planner_itinerary_generated` — AI created itinerary
- `planner_handoff_requested` — User clicked "Get Full Quote"
- `planner_lead_created` — Lead submitted

---

### Module 7: WhatsApp Integration (P2)

**Why:** Deliver itinerary + booking confirmations via WhatsApp (familiar for Indians).

**Files to Create:**
```
lib/whatsapp/client.ts            ← WhatsApp Business API client
lib/whatsapp/templates.ts         ← Message templates
components/WhatsAppShare.tsx      ← Share itinerary button
```

**Integration Point:**
- After handoff, send itinerary to user's WhatsApp
- Booking confirmation via WhatsApp
- Trip reminder via WhatsApp

---

### Module 8: Voice Interface (P3)

**Why:** Hands-free planning ("Hey GoRASA, plan a Goa trip").

**Files to Create:**
```
hooks/useSpeechRecognition.ts     ← Web Speech API wrapper
components/VoiceInputButton.tsx   ← Microphone button with waveform
components/VoiceWaveform.tsx      ← Animated waveform
```

**Integration Point:**
- Add mic button to ChatInterface
- Voice input → text → existing flow
- Voice output already exists (TTS in ConciergeChat)
