### INTEGRATE as Features (high-value unused code)

#### 1. EscalationFlow → Support Page Enhancement
**What:** Modal with 4 escalation options (Create Ticket, WhatsApp, Call, Email)
**Where:** Add to `/support` page as a "Contact Options" button
**Value:** Users currently only see ticket creation. This adds WhatsApp, phone, and email channels.
**Effort:** Low — import and render conditionally on support page.

#### 2. UnifiedChat → Replace/Enhance ChatInterface
**What:** Full chat UI with intent classification, routing to flights/hotels/planner, escalation
**Where:** Could replace existing `ChatInterface.tsx` or be offered as an alternative mode
**Value:** Current chat is basic. This adds AI-powered routing and smart responses.
**Effort:** Medium — needs integration testing with existing support flow.

#### 3. PlannerHandoff → Holiday Planner Flow
**What:** "Start Planning" card shown when AI detects holiday intent
**Where:** Add to support chat or homepage when user expresses travel interest
**Value:** Smooth transition from browsing/chat to the AI planner.
**Effort:** Low — simple component, just needs to be rendered at the right trigger point.

#### 4. QuickReplies + TypingIndicator → Chat UI Components
**What:** Reusable quick reply buttons and typing animation
**Where:** Use in ChatInterface, UnifiedChat, or any future chat component
**Value:** Improves chat UX with interactive buttons and visual feedback.
**Effort:** Low — drop-in components.

#### 5. Analytics Events → User Tracking
**What:** Event tracking system with predefined events (chat, planner, tickets, filters, navigation)
**Where:** Wire into key user actions across the app
**Value:** Currently zero analytics. This enables measuring user behavior.
**Effort:** Medium — need to add `track()` calls to key interactions.

#### 6. Feature Flags → Environment-Based Toggles
**What:** Feature flag system using env vars (AI_PLANNER, SUPPORT_CHAT, TICKETS, etc.)
**Where:** Wrap features in `<FeatureFlag flag="TICKETS_ENABLED">` to toggle them
**Value:** Enables gradual rollout and env-based feature control.
**Effort:** Low — already has component + config, just needs env vars set.
